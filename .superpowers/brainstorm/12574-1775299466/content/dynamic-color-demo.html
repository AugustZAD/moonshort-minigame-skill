场景 A：独立游戏
用户直接打开游戏 URL：
game.html?theme=combat

→ 使用 7 套静态 THEMES 之一
→ 不需要封面图
→ 不触发 K-Means
THEMES['combat'] → 直接写入 CSS 变量
场景 B：Episode 剧情嵌入
构建脚本注入剧情上下文：
window.__EPISODE_CTX__ = {
  coverImage: "https://..../ep8-cover.jpg",
  theme: "combat" // fallback
}

→ 有 coverImage? 触发动态取色
→ 从封面图提取 5 色调色板
→ 游戏颜色自动匹配故事氛围
extractPalette(cover) → 动态调色板 → CSS 变量
核心逻辑：两条路径最终都写入同一套 CSS 变量（--primary, --bg, --primary-10 等）。游戏代码不关心颜色从哪来，它只读 CSS 变量。
// 游戏启动时的统一入口 async function resolveTheme() { // 优先级 1: 封面图动态提色 if (CTX.coverImage) { const palette = await extractFromCover(CTX.coverImage); if (palette) return paletteToTheme(palette); // 质检不通过? 继续往下走 fallback } // 优先级 2: 静态预设 return THEMES[CTX.theme || 'combat']; } const theme = await resolveTheme(); applyTheme(theme); // 写入 CSS 变量，游戏不感知来源

为什么要「年轻感评分」? 不是所有颜色都适合游戏 UI。封面图里可能有土黄色的沙漠、灰色的建筑——这些颜色用在按钮上很丑。评分公式确保：

最高优先级
紫色 ×1.22
高优先级
蓝色 ×1.15
中等
土黄 ×0.65
降权，避免脏色
Fallback 瀑布（单色封面兜底）：
① 标准贪心从 mids 选色 → ② 不足 2 色? 从 darks 中 S>30% 的提亮 → ③ 仍不足? 数学生成互补色（色相旋转 150°）→ ④ 全部失败? fallback 到静态 THEMES 预设

质检硬门槛：primary.L 40-75%, primary.S >60%, surface.L >80%。不通过直接回退静态主题。

封面图层是可选的。
• 独立游戏（场景 A）：不加封面图层，纯色背景
• Episode 嵌入（场景 B）：有 CTX.coverImage 时，插入 <img id="cover-layer" src="...">
• 封面图层和动态取色是独立的——可以只用取色不用图层，也可以只用图层不取色
